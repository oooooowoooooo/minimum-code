# Good First Issues

Use this list to create beginner-friendly GitHub issues. Each issue should include a small scope, clear acceptance criteria, and a verification command.

| Title | Labels | Acceptance criteria |
| --- | --- | --- |
| Add Python async knowledge point | `good first issue`, `content`, `python` | Add one JSON knowledge point with quiz and game content; `python verify_all.py` passes. |
| Add TypeScript generic quiz | `good first issue`, `content`, `typescript` | Add or improve one generic-related quiz; frontend renders it correctly. |
| Add FastAPI dependency injection card | `good first issue`, `source-reading`, `fastapi` | Add a concise card explaining `Depends()` and request-scoped resolution. |
| Add LangChain Runnable dissection card | `good first issue`, `source-reading`, `langchain` | Explain `Runnable` composition and include one exercise. |
| Add screenshot to README | `good first issue`, `docs`, `design` | Replace the preview image with a current product screenshot. |
| Add CI workflow polish | `good first issue`, `ci` | Improve cache usage or job names without changing verification coverage. |
| Refactor backend module registry into JSON | `good first issue`, `backend` | Keep module API output unchanged while moving registry data to JSON. |
| Add module search test | `good first issue`, `test`, `frontend` | Add one focused Vitest case for module search or filtering behavior. |
| Add dark mode polish | `good first issue`, `frontend`, `design` | Improve contrast and hover states without adding a new color theme. |
| Add roadmap page | `good first issue`, `docs`, `product` | Turn `ROADMAP.md` into a linked product page or docs entry. |
