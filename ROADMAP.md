# Roadmap

`minimum-code` is moving from a broad programming-learning prototype toward an AI-era source-code reading and architecture training platform.

## Current Position

```text
From: broad programming education content
To: source-code reading and architecture training for AI-era developers
```

The highest ROI path is to make the repository credible, runnable, visible, and easy to contribute to before adding more content.

## 7-Day Launch Plan

### Day 1: Trust

- Keep README free of merge conflict markers.
- Remove unverifiable hard-coded test-count claims from badges.
- Add GitHub Actions CI for frontend tests, backend tests, and data verification.
- Add a homepage screenshot or demo GIF.
- Set repository topics on GitHub.

### Days 2-3: Maintainability

- Keep `main.py` focused on FastAPI app wiring.
- Keep route handlers thin.
- Keep module registry and quiz content in JSON.
- Keep source extraction and content lookup in services.
- Add focused schema validation where content crosses an API boundary.

### Days 4-5: Killer Path

Primary path:

```text
7 days to understand Agent engineering source code:
LangChain + FastAPI + multi-agent workflow patterns
```

Deliverables:

- Day-by-day path page.
- FastAPI dependency injection card.
- LangChain Runnable card.
- Agent tool-calling card.
- Review questions that test architecture judgment, not syntax memory.

### Days 6-7: Public Launch

- Publish the hosted demo.
- Add a 30-second demo GIF.
- Open the first 10 good-first issues.
- Publish a launch article:
  `AI 时代，为什么程序员应该少写代码，多读好代码`
